# Inspectability Trial Operator Handoff Checklist

Use this checklist to run the next real Hermes/Mission Control inspectability review without mistaking templates for proof. It sequences the existing packet, debrief, rubric, evidence index, and promotion card into one operator handoff.

## Before the trial

- [ ] Confirm the target workflow, repo/app/screen, and success threshold are written in the trial packet.
- [ ] Open the prototype checklist/builder and capture its starting state or exported markdown.
- [ ] Create a fresh evidence folder or note with links for logs, screenshots, recordings, commits, and cost/runtime notes.
- [ ] Review the evidence quality rubric before starting so weak proof is labeled weak immediately.

## During the trial

- [ ] Save raw transcript/log output before summarizing it.
- [ ] Capture every operator intervention, restart, rollback, or manual workaround.
- [ ] Link commits, PRs, screenshots, or command outputs next to the exact task step they support.
- [ ] Note any claim that should stay narrowed, pilot-only, or explicitly unvalidated.

## After the trial

- [ ] Fill the post-trial debrief while details are still fresh.
- [ ] Complete the evidence quality rubric and mark unsupported or low-quality claims.
- [ ] Fill the trial evidence index, mapping each proof item to exact README/prototype/skill claims.
- [ ] Complete the promotion decision card with promote / pilot-only / iterate / hold and patch targets.
- [ ] Patch package/prototype/skill wording only after the evidence index and promotion card agree.
